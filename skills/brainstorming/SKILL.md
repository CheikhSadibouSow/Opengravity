# Skill: Brainstorming (Design Thinking)

## 💡 Objectif
Ne jamais commencer à coder avant d'avoir une vision claire et validée par l'utilisateur.

## 🛠️ Processus
1. **Écouter** : Analyser la demande brute de l'utilisateur.
2. **Questionner** : Poser 3 à 5 questions ciblées pour lever les ambiguïtés (edge cases, stack technique, UI/UX).
3. **Proposer** : Rédiger une spécification technique courte sous forme de "Specs".
4. **Valider** : Attendre le "OK" de l'utilisateur avant de passer à l'étape suivante.

## 📝 Format de sortie
L'agent doit produire un document `SPECIFICATION.md` contenant :
- Résumé du projet
- Fonctionnalités clés
- Choix techniques
- Schéma de données (si applicable)
